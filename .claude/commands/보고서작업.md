📊 **보고서작업 (Research Report Management)**

**🎯 목적**: 연구 진행상황과 결과를 다양한 형태의 보고서로 체계적 생성 및 관리

**📚 컨텍스트 자동 로딩:**
- research/experiments/completed/ 확인 (완료된 실험들)
- research/experiments/active/ 확인 (진행 중인 실험들)
- docs/CURRENT/research-log.md 확인 (전체 연구 이력)
- project_rules.md 확인 (프로젝트 맥락)

**📋 사용법:**
```
/보고서작업 [보고서타입] [버전] [옵션]
```

**⚡ 4단계 보고서 생성 프로세스:**

## **2. 🔍 자동 데이터 수집 및 분석**

**실험 데이터 자동 집계:**
```python
def collect_experiment_data():
    experiments = {
        "completed": scan_directory("research/experiments/completed/"),
        "active": scan_directory("research/experiments/active/"),
        "archived": scan_directory("research/experiments/archived/")
    }
    
    metrics = {
        "total_experiments": len(experiments["completed"]) + len(experiments["active"]),
        "success_rate": calculate_success_rate(experiments["completed"]),
        "average_duration": calculate_avg_duration(experiments["completed"]),
        "top_performing": identify_top_experiments(experiments["completed"])
    }
    
    return experiments, metrics
```

**성과 지표 자동 추출:**
- 각 실험의 `results/metrics/` 디렉토리 스캔
- `FINAL/key_findings.json` 구조화된 데이터 수집
- 시간별, 카테고리별 성과 트렌드 분석
- 실패 패턴 및 성공 요인 식별

## **⚙️ 고급 옵션**

### 보고서 유형별 옵션
```
# 진행상황 보고서
--period TIMEFRAME        : 특정 기간 (last-week, last-month, custom)
--include-active         : 진행 중인 실험 포함
--trend-analysis         : 트렌드 분석 포함

# 분석 보고서  
--focus-experiment ID    : 특정 실험 집중 분석
--comparative           : 여러 실험 비교 분석
--deep-dive             : 심화 분석 모드

# 최종 보고서
--include-all           : 모든 실험 데이터 포함  
--timeline              : 시간순 전개 형식
--executive-summary     : 경영진용 요약본 추가
```

### 출력 형식 옵션
```
--format pdf/docx/slides : 특정 형식으로 변환
--template STYLE        : 템플릿 스타일 (academic/business/technical)
--auto-export          : 자동 외부 형식 변환
--email-ready          : 이메일 발송 가능 형식
```
